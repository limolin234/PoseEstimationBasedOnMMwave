## 二：雷达数据解析

我们先在PC端推理验证：直接使用 Radar Data Output 模式获取结构化的点云数据，再写python脚本来实现  

### 步骤

读取原始字节流+帧头校验（筛选有效帧）+按规则拆分字节（拆成单个数据单元）+字节转可读数值（核心）+提取有效数据（过滤 + 精简）  

### 优化方向：流式处理保证实时，优化串口缓冲逻辑

## 三：数据预处理

步骤2：归一化

步骤4：异常值过滤  

## 四：转热图张量
对于5维数据进行处理与输入模型计算会带来不必要的消耗，因此采用常用的在输入模型之前现将数据转化为热图：笛卡尔坐标转极坐标公式
将毫米波点云转换为 Range-Azimuth Map (RAM)  
    输入: points (N, 5) -> [x, y, z, v, snr]  
    输出: ram_map (range_bins, angle_bins) -> 每个 cell 存最大 SNR 或点数  
保留了设置get_stacked_ram_input堆叠多帧输入  
### 优化方向：时序模型
## 五：点云特征提取

构造了Net神经网络结构如下： 
我们在这里讲人体姿态识别看做一个分类任务，而如果要做到精度更高的骨架的输出则是回归任务，这里参考了了经典的U-net思路，分为编码器（下采样）、瓶颈层（特征融合）、解码器（上采样） 三大模块，核心是通过"下采样提取特征+上采样恢复尺寸"实现精准分割。

### Backbone

输出feat：得到特征向量：融合距离 / 方位角 / SNR，高维通用 ；适合分类  

### Heatmaps

（解码器）输出heatmaps：  14个骨架关键点  

### Heights

（高度分支）输出heights： 关键点的高度  

### 所以考虑了几种方案

1：backbone直接输入下一个姿态分类器  
2：精度更高的话将关键点与高度拼接在特征向量中再分类  
3：通过精准的骨架点直接生成骨架  

### 优化方向：迁移学习 多尺度特征

## 六：姿态分类

ClassifierNet  
feat_type="keypoint"：输入 3D 关键点展平特征（14×3=42 维），轻量级全连接层分类；  
feat_type="backbone"：输入 Net 网络骨干的卷积特征（64×8×4=2048 维），更复杂的全连接层分类。  
两个神经网络采用分阶段训练  

## 七：重构流程
### 重构路线图
模型加载层：将 .pth 权重转换为 NumPy 字典，或用 ONNX 替代。  
算子映射层：建立 torch.nn.functional 到 numpy 的映射表（卷积、归一化、激活函数）。  
数据流层：移除所有 .cuda(), .detach(), .numpy() 转换，统一使用 np.ndarray。  
后处理层：将 Argmax/Softmax 等操作改为 NumPy 实现。
