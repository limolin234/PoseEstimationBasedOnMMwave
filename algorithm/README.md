## 二：雷达数据解析
我们先在PC端推理验证：直接使用 Radar Data Output 模式获取结构化的点云数据，再写python脚本来实现  
### 步骤：读取原始字节流+帧头校验（筛选有效帧）+按规则拆分字节（拆成单个数据单元）+字节转可读数值（核心）+提取有效数据（过滤 + 精简）  
### 需要的数据
帧结构规则：
帧头：有效数据帧的起始字节（比如加特兰 AK 帧头是0x55AA，占 2 字节）；  
帧长度：每帧总字节数（或数据段长度）；  
帧尾 / 校验：可选，验证帧完整性（比如 CRC 校验码）。  
数据段排列规则：  
每个点的字节组成（比如 X 占 4 字节 + Y 占 4 字节 + Z 占 4 字节 + SNR 占 2 字节，共 14 字节 / 点）；  
数据段在帧中的位置（比如帧头后第 0 字节开始是点云数据）。  
数据类型：  
数值的存储类型（X/Y/Z 是float32（4 字节），SNR 是int16（2 字节））；  
字节序（小端 /little-endian，加特兰雷达默认小端）。  
有效数据范围：  
哪些数据是你需要的（比如只取 X/Y/Z，丢弃 SNR / 速度）；  
噪声阈值（比如距离 < 0.5 米的点是无效噪声）。  
硬件读取规则：  
串口参数（波特率 921600、8 位数据位、1 位停止位）；  
FPGA 端 UART 读取方式（比如 Pynq 的/dev/ttyPS0设备）  
### 优化方向：流式处理保证实时

## 四：点云特征提取
构造了Net神经网络结构如下：  
### Backbone 
输出feat：得到特征向量：融合距离 / 方位角 / SNR，高维通用 ；适合分类  
### Heatmaps
（解码器）输出heatmaps：  14个骨架关键点  
### Heights
（高度分支）输出heights： 关键点的高度  
### 所以考虑了几种方案：
1：backbone直接输入下一个姿态分类器  
2：精度更高的话将关键点与高度拼接在特征向量中再分类  
3：通过精准的骨架点直接生成骨架  
### 优化方向：迁移学习 多尺度特征
## 五：姿态分类
ClassifierNet
feat_type="keypoint"：输入 3D 关键点展平特征（14×3=42 维），轻量级全连接层分类；
feat_type="backbone"：输入 Net 网络骨干的卷积特征（64×8×4=2048 维），更复杂的全连接层分类。
两个神经网络采用分阶段训练
